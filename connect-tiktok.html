Connecter TikTok — sans terminal

Tout se passe dans cette page + GitHub. Pas de Python, pas de localhost. La seule action imposée par TikTok : te connecter au compte et cliquer Authorize. AWEMA fait le reste (échange sécurisé en arrière-plan).

⏱️ ~10 min la 1ʳᵉ fois · puis 30 s par compte · 100 % navigateur

1 Enregistrer les clés de ton app TikTok une fois

Depuis developers.tiktok.com (ton app → Manage apps), récupère Client key et Client secret. Il te faut aussi un PAT GitHub (fine-grained, portée Variables : Read and write) — en créer un ↗. Colle-les ici : AWEMA les enregistre en Secrets GitHub (jamais dans le dépôt).

Ces 3 valeurs deviennent les Secrets TIKTOK_CLIENT_KEY, TIKTOK_CLIENT_SECRET, TIKTOK_PAT. Si l'écriture auto n'est pas dispo, AWEMA ouvre le bon écran GitHub avec la valeur déjà copiée (un collage).

2 Autoriser l'adresse de retour dans l'app TikTok une fois

TikTok doit savoir vers quelle page te ramener après ton autorisation. Ouvre la section Login Kit, repère le champ Redirect URI (= adresse de retour) et colle-y exactement cette adresse :

(adresse en cours de chargement…)
C'est l'URL de oauth.html sur ton site. TikTok accepte les adresses HTTPS — plus besoin de localhost.

3 Connecter un compte TikTok

Connecte-toi d'abord, dans ce navigateur, au compte TikTok que tu veux ajouter. Puis clique :

Tu arrives sur TikTok → Authorize → retour automatique sur AWEMA, qui finalise (échange du code par GitHub Actions). Pour plusieurs comptes : change de compte TikTok connecté, puis reviens ici et recommence.

4 Récupérer les stats

Une fois tes comptes connectés, lance la synchro : leurs vraies métriques arrivent dans le dashboard. Ensuite c'est automatique chaque lundi.
